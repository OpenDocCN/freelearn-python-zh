Files, Printers, and Other Dialogs

Any application with a GUI is created to produce operations. To do this in a user-friendly manner, it needs to have some basic functionality, such as the ability to open and save files, inform the user when something is processing, or if an error appears. To implement this in an application, we can use various dialog windows that open and save files, and inform the user that an event has occurred. Additionally, if a user's tasks relate to working with text, they may want to print the text in paper form or to a file, such as a PDF. In this chapter, we will be covering the basic dialog windows provided by the Qt library, and we will demonstrate some simple print implementations in the app, including the following:

  • Dialog windows
  • Message boxes
  • Font and color dialog windows
  • Files
  • Printers

Dialog windows

Dialog windows are most commonly used to realize communications between the user and application processes. These dialog windows can open and save files, show error messages, allow the user to change a printer's settings before printing, choose colors and fonts, and more. The Qt library provides the most popular dialog windows for an application. A dialog window is a kind of widget that can be modal or modeless. Modal dialog windows block input from other application windows when they are visualized. In comparison to this, modeless dialog windows provide their functionality independently from the windows of the application. Here, we will introduce the base class for dialog windows, that is, QDialog. Additionally, we'll examine additional classes, such as QDialogButtonBox, which constructs button boxes.

QDialog

This class is the base class of the dialog window used with Qt. Almost all functional dialog windows inherit this class. The declaration syntax is as follows:

dialog = QtWidgets.QDialog()

The QDialog class enhances functionality with the functions described in the following sections.

set

These functions set parameters or properties to the dialog window:

dialog.setModal(bool): If the parameter is True, it sets this dialog window as modal.

dialog.setResult(int): This sets the dialog window's result code to the code specified in the parameter.

dialog.setSizeGripEnabled(bool): If the parameter is True, it sets the resize handle of the QSizeGrip type for resizing top-level windows.

dialog.setVisible(bool): If the parameter is True, it sets this dialog window as visible.

is

This function returns a Boolean value (bool) related to this dialog window's state:

dialog.isSizeGripEnabled(): This returns True if the size grip is enabled for this dialog window.

functional

These functions return the current values of the dialog window, inform you about changes to functionality, and more:

dialog.accept(): This accepts the dialog and sets the result code to Accepted with respect to hiding the modal dialog window.

dialog.done(int): This closes the dialog and sets the result code with respect to the code specified in the parameter.

dialog.exec(): This displays and executes the modal dialog and returns the dialog result code.

dialog.exec_(): This displays and executes the modal dialog and returns the dialog result code.

dialog.open(): This opens and shows the modal dialog window.

dialog.reject(): This rejects the dialog and sets the result code to Rejected when the modal dialog is hidden.

dialog.result(): This returns the result code of the modal dialog, such as Accepted or Rejected.

events

These functions are related to events, such as event handlers:

dialog.closeEvent(QtGui.QCloseEvent): This event handler is called with the event parameter when it receives a dialog window close request.

dialog.contextMenuEvent(QtGui.QContextMenuEvent): This event handler receives dialog window context menu events.

dialog.eventFilter(QtCore.QObject, QtCore.QEvent): This event handler filters events if the dialog window is installed as an event filter for the object (the first parameter).

dialog.keyPressEvent(QtGui.QKeyEvent): This event handler receives key press events for the dialog window with the event passed in the parameter.

dialog.resizeEvent(QtGui.QResizeEvent): This event handler receives resize events for the dialog window with the event passed in the parameter.

dialog.showEvent(QtGui.QShowEvent): This event handler receives show events for the dialog window with the event passed in the parameter.

signals

These are the available signals of the QDialog class:

dialog.accepted(): This signal is emitted when the dialog has been accepted by the user, or by calling the accept() and done() functions.

dialog.finished(int): This signal is emitted when the result code of the dialog has been set by the user, or by calling the accept(), done(), and reject() functions.

dialog.rejected(): This signal is emitted when the dialog has been rejected by the user, or by calling the reject() and done() functions.

Message boxes

Message boxes are used to inform the user about events that occur in the application. These can be windows with messages, errors, and other events. These widgets are dialog windows and they provide functionality in relation to these events.

QMessageBox

This class implements the communication between the user and the application by asking questions and receiving answers relating to the process. The box is constructed with the modal dialog window representation. The declaration syntax of this class is as follows:

mesage_box = QtWidgets.QMessageBox()

The QMessageBox class inherits the QDialog class and improves functionality with the functions described in the following sections.

add

These functions add elements to the message box:

message_box.addButton(QtWidgets.QMessageBox.StandardButton): This adds a standard button to the message box and returns a push button.

message_box.addButton(QtWidgets.QAbstractButton, QtWidgets.QMessageBox.ButtonRole): This adds a button (the first parameter) to the message box with a role (the second parameter). The button roles are as follows:

  • QtWidgets.QMessageBox.Ok: The OK button with AcceptRole.
  • QtWidgets.QMessageBox.Open: The Open button with AcceptRole.
  • QtWidgets.QMessageBox.Save: The Save button with AcceptRole.
  • QtWidgets.QMessageBox.Cancel: The Cancel button with RejectRole.
  • QtWidgets.QMessageBox.Close: The Close button with RejectRole.
  • QtWidgets.QMessageBox.Discard: The Discard or Don't Save (platform-dependent) button with DestructiveRole.
  • QtWidgets.QMessageBox.Apply: The Apply button with ApplyRole.
  • QtWidgets.QMessageBox.Reset: The Reset button with ResetRole.
  • QtWidgets.QMessageBox.RestoreDefaults: The Restore Defaults button with ResetRole.
  • QtWidgets.QMessageBox.Help: The Help button with HelpRole.
  • QtWidgets.QMessageBox.SaveAll: The Save All button with AcceptRole.
  • QtWidgets.QMessageBox.Yes: The Yes button with YesRole.
  • QtWidgets.QMessageBox.YesToAll: The Yes To All button with YesRole.
  • QtWidgets.QMessageBox.No: The No button with NoRole.
  • QtWidgets.QMessageBox.NoToAll: The No To All button with NoRole.
  • QtWidgets.QMessageBox.Abort: The Abort button with RejectRole.
  • QtWidgets.QMessageBox.Retry: The Retry button with AcceptRole.
  • QtWidgets.QMessageBox.Ignore: The Ignore button with AcceptRole.
  • QtWidgets.QMessageBox.NoButton: This is an invalid button.

message_box.addButton(str, QtWidgets.QMessageBox.ButtonRole): This adds a button with text (the first parameter) to the message box with a role (the second parameter), and returns a push button.

set

These functions set the parameters or properties to the message box:

message_box.setCheckBox(QtWidgets.QCheckBox): This sets the checkbox specified in the parameter in the message box dialog.

message_box.setDefaultButton(QtWidgets.QPushButton): This sets the default button specified in the parameter for the message box.

message_box.setDefaultButton(QtWidgets.QMessageBox.StandardButton): This sets the default button specified in the parameter for the message box.

message_box.setDetailedText(str): This sets the detailed text specified in the parameter for the message box. The detailed text is plain text, which will be displayed in the details area.

message_box.setEscapeButton(QtWidgets.QAbstractButton): This sets the button specified in the parameter, which will be activated when the Esc key is pressed.

message_box.setEscapeButton(QtWidgets.QMessageBox.StandardButton): This sets the button specified in the parameter, which will be activated when the Esc key is pressed.

message_box.setIcon(QtWidgets.QMessageBox.Icon): This sets the icon that will be used with the message box. The available icons are as follows:

  • QtWidgets.QMessageBox.NoIcon—0: No Icon is displayed for the message box.
  • QtWidgets.QMessageBox.Information —1: Icon displaying information sign.
  • QtWidgets.QMessageBox.Warning —2: Icon displaying a warning sign.
  • QtWidgets.QMessageBox.Critical—3: Icon displaying critical problem sign.
  • QtWidgets.QMessageBox.Question—4: Icon displaying question sign.

message_box.setIconPixmap(QtGui.QPixmap): This sets the pixmap specified in the parameter for the icon of the message box.

message_box.setInformativeText(str): This sets the informative text specified in the parameter, which will be a full description of the message in the message box.

message_box.setStandardButtons(QtWidgets.QMessageBox.StandardButtons): This sets standard buttons specified in the parameter, which will be used in the message box.

message_box.setText("Message Box Text"): This sets the text specified in the parameter, which will be displayed in the message box.

message_box.setTextFormat(QtCore.Qt.TextFormat): This sets the text format specified in the parameter for the text that will be displayed in the message box.

message_box.setTextInteractionFlags(QtCore.Qt.TextInteractionFlags): This sets the text interaction flags specified in the parameter, which will describe the label interaction with user input.

message_box.setWindowModality(QtCore.Qt.WindowModality): This sets the modality specified in the parameter for the message box.

message_box.setWindowTitle(str): This sets the title specified in the parameter for this dialog window message box.

functional

These functions return the current values of the message box, inform you about changes to functionality, and more:

message_box.about(QtWidgets.QWidget, str, str): This displays a simple box with a title (the second parameter) and text (the third parameter) about the first parameter.

message_box.aboutQt(QWidget, str): This displays a simple box with a title (the second parameter) about Qt.

message_box.button(QtWidgets.StandardButton): This returns the button of the QtWidgets.QAbstractButton type corresponding to the button specified in the parameter.

message_box.buttonRole(QtWidgets.QAbstractButton): This returns the button role of the QtWidgets.QMessageBox.ButtonRole type for the button specified in the parameter.

message_box.buttons(): This returns all the buttons added to the message box as a list.

message_box.checkBox(): This returns the checkbox of the QtWidgets.QCheckBox type, which is available on the message box; otherwise, it returns 0.

message_box.clickedButton(): This returns the button of the QtWidgets.QAbstractButton type that was clicked on.

message_box.critical(QtWidgets.QWidget, str, str, QtWidgets.QMessageBox.StandardButton, QtWidgets.QMessageBox.StandardButton): This opens a critical message box with a title (the second parameter), some text (the third parameter) in front of the widget specified (the first parameter), and the buttons specified in the fourth and fifth parameters.

message_box.defaultButton(): This returns the default button of the QtWidgets.QPushButton type for the message box.

message_box.detailedText(): This returns the plain text displayed in the details area.

message_box.escapeButton(): This returns the button that will be activated when the user presses the Esc key.

message_box.icon(): This returns the icon of the QtWidgets.QMessageBox.Icon type used with the message box.

message_box.iconPixmap(): This returns the current icon pixmap of the QtGui.QPixmap type.

message_box.information(QtWidgets.QWidget, str, str, QtWidgets.QMessageBox.StandardButton, QtWidgets.QMessageBox.StandardButton): This opens an information message box with a title (the second parameter), some text (the third parameter) in front of the widget specified (the first parameter), and the buttons specified in the fourth and fifth parameters.

message_box.informativeText(): This returns informative text, such as a full description of the message in the message box.

message_box.open(QtCore.QObject, str): This connects a signal to a slot with a receiver (the first parameter) and a member (the second parameter).

message_box.question(QtWidgets.QWidget, str, str, QtWidgets.QMessageBox.StandardButton, QtWidgets.QMessageBox.StandardButton): This opens a question message box with a title (the second parameter), some text (the third parameter) in front of the widget specified (the first parameter), and the buttons specified in the fourth and fifth parameters.

message_box.removeButton(QtWidgets.QAbstractButton): This removes the button specified in the parameter from the message box.

message_box.standardButton(QtWidgets.QAbstractButton): This returns the button of the QtWidgets.QMessageBox.StandardButton type corresponding to the button specified in the parameter.

message_box.standardButtons(): This returns the standard buttons of the QtWidgets.QMessageBox.StandardButtons type that are used with the message box.

message_box.standardIcon(QtWidgets.QMessageBox.Icon): This returns the pixmap of the QtGui.QPixmap type, which is used for the standard icon of the message box.

message_box.text(): This returns the text that is displayed in the message box.

message_box.textFormat(): This returns the format of the QtCore.Qt.TextFormat type, which is used for the text in the message box.

message_box.textInteractionFlags(): This returns the flags of the QtCore.Qt.TextInteractionFlags type, which are used for the text interaction of the message box.

message_box.warning(QtWidgets.QWidget, str, str, QtWidgets.QMessageBox.StandardButton, QtWidgets.QMessageBox.StandardButton): This opens a warning message box with a title (the second parameter), some text (the third parameter) in front of the widget specified (the first parameter), and the buttons specified in the fourth and fifth parameters.

events

These functions are related to events, such as event handlers:

message_box.changeEvent(QtCore.QEvent): This handles the state changes of the message box.

message_box.closeEvent(QtGui.QCloseEvent): This event handler is called with the event parameter when it receives a close request for a message box from the window system.

message_box.event(QtCore.QEvent): This receives events sent to the object and should return True if the events are recognized and processed.

message_box.keyPressEvent(QtGui.QKeyEvent): This event handler receives keypress events for the message box with the event passed in the parameter.

message_box.resizeEvent(QtGui.QResizeEvent): This event handler receives resize events for the message box with the event passed in the parameter.

message_box.showEvent(QtGui.QShowEvent): This event handler receives show events for the message box with the event passed in the parameter.

signals

The following signal is available with the QMessageBox class:

message_box.buttonClicked(QtWidgets.QAbstractButton): This signal is emitted when the button is clicked on inside the message box.

QErrorMessage

This class displays the error message dialog window. Errors can occur in an application, and this window informs the user about these events. The declaration syntax of this class is as follows:

error_message = QtWidgets.QErrorMessage()

QErrorMessage inherits the QDialog class and enhances its functionality with the functions described in the following sections.

functional

These functions return the current values of the error message dialog, inform you about changes to functionality, and more:

error_message.qtHandler(): This returns an object of the QtWidgets.QErrorMessage type, which outputs the default Qt messages.

error_message.showMessage(str): This shows the message specified in the parameter.

error_message.showMessage(str, str): This shows a message (the first parameter) with the requested message type (the second parameter).

events

These functions are related to events, such as event handlers:

error_message.changeEvent(QtCore.QEvent): This handles the state changes of the error message dialog.

Font and color dialogs

The Qt framework provides standard dialog windows for selecting fonts and colors in an application. They can be used to change the font of some text or the color of an element.

QFontDialog

This class realizes the dialog window for the font selection. The selected font can be used for text in text fields, labels, and for other application elements. The declaration syntax of this class is as follows:

font_dialog = QtWidgets.QFontDialog()

QFontDialog inherits the QDialog class and enhances its functionality with functions described in the following sections:

set

These functions set the parameters or properties of the font dialog:

font_dialog.setCurrentFont(QtGui.QFont): This sets the font specified in the parameter to be highlighted in the font dialog.

font_dialog.setOption(QtWidgets.QFontDialog.FontDialogOption, bool): If the second parameter is True, then the font dialog option is set. The available options are as follows:

  • QtWidgets.QFontDialog.NoButtons: Font dialog without buttons
  • QtWidgets.QFontDialog.DontUseNativeDialog: Font dialog as Qt standard
  • QtWidgets.QFontDialog.ScalableFonts: Font dialog with scalable fonts
  • QtWidgets.QFontDialog.NonScalableFonts: Font dialog with non-scalable fonts
  • QtWidgets.QFontDialog.MonospacedFonts: Font dialog with monospaced fonts
  • QtWidgets.QFontDialog.ProportionalFonts: Font dialog with proportional fonts

font_dialog.setOptions(QtWidgets.QFontDialog.FontDialogOptions): This sets the options that will be used with the font dialog.

functional

These functions return the current values of the font dialog, inform you about changes to functionality, and more:

font_dialog.currentFont(): This returns the current font of the QtGui.QFont type.

font_dialog.getFont(QtWidgets.QWidget): This returns the font and executes a modal font dialog, with the parent of the dialog specified.

font_dialog.getFont(QtGui.QFont, QtWidgets.QWidget, str, QtWidgets.QFontDialog.FontDialogOptions): This returns the font and executes a modal font dialog. If the user clicks on the OK button, it returns the selected font. If the user clicks on Cancel, then the initial font (the first parameter) will be returned. It's constructed with the parent widget (the second parameter), the title (the third parameter) of the dialog window, and options (the fourth parameter).

font_dialog.open(QtCore.QObject, str): This connects a signal to a slot with a receiver (the first parameter) and a member (the second parameter).

font_dialog.options(): This returns the options of the QtWidgets.QFontDialog.FontDialogOptions type that is used for the font dialog.

font_dialog.selectedFont(): This returns the selected font of the QtGui.QFont type by clicking on the OK button, or by accepting in another manner.

font_dialog.testOption(QtWidgets.QFontDialog.FontDialogOption): This returns True if the font dialog option specified in the parameter is enabled; otherwise, it returns False.

events

This function is related to events:

font_dialog.changeEvent(QtCore.QEvent): This handles the state changes of the font dialog.

signals

The available signals of the QFontDialog class are as follows:

font_dialog.currentFontChanged(QtGui.QFont): This signal is emitted when the current font for the font dialog changes, and the new font is passed in the parameter.

font_dialog.fontSelected(QtGui.QFont): This signal is emitted when the font is selected in the font dialog, with the selected font passed in the parameter.

color_dialog.setOptions(QtWidgets.QFontDialog.FontDialogOptions): This sets the options that will be used with the color dialog.

color_dialog.setStandardColor(int, QtGui.QColor): This sets the standard color (the second parameter) at the index (the first parameter).

functional

color_dialog.customColor(int): This returns the custom color of the QtGui.QColor type at the index specified in the parameter.

color_dialog.customCount(): This returns the number of custom colors that are supported by the color dialog.

color_dialog.getFont(QtGui.QColor, QtWidgets.QWidget, str, QtWidgets.QColorDialog.ColorDialogOptions): This returns the color and executes a modal color dialog. When the user clicks on the Ok button, it returns the selected color. If the user clicks on Cancel, then the initial color (the first parameter) will be returned. It is constructed with the parent widget (the second parameter), the title (the third parameter) of the dialog window, and options (the fourth parameter).

color_dialog.open(QtCore.QObject, str): This connects a signal to a slot with a receiver (the first parameter) and a member (the second parameter).

color_dialog.options(): This returns the options of the QtWidgets.QColorDialog.ColorDialogOptions type used for this color dialog.

color_dialog.selectedColor(): This returns the selected color of the QtGui.QColor type by clicking or the Ok button or by accepting in another manner.

color_dialog.standardColor(int): This returns the standard color of the QtGui.QColor type at the index specified in the parameter.

color_dialog.testOption(QtWidgets.QColorDialog.ColorDialogOption): This returns True if the color dialog option specified in the parameter is enabled; otherwise, it returns False.

events

Files

In this section, we will describe the dialog windows that provide the functionality to open or save files in the application. Additionally, we will cover reading and writing to files. Almost all GUI applications operate or have the ability to manipulate various files. In the Qt library, this file dialog window can realize the QFileDialog class.

QFileDialog

This class creates a dialog window for basic file manipulation in the application, such as opening a file, saving a file with a specified name, or opening a directory containing files. The declaration syntax of this class in the application is as follows:

file_dialog = QtWidgets.QFileDialog()

QFileDialog inherits the QDialog class and enhances its functionality with the functions described in the following sections.

set

These functions set the parameters or properties for the file dialog:

file_dialog.setAcceptMode(QtWidgets.QFileDialog.AcceptMode): This sets the accept mode of the dialog as specified in the parameter, which explains that the file dialog is for opening or saving files. The available modes are as follows:

  • QtWidgets.QFileDialog.AcceptOpen—0: Dialog for opening files.
  • QtWidgets.QFileDialog.AcceptSave—1: Dialog for saving files.

file_dialog.setDefaultSuffix(str): This sets the default suffix, specified in the parameter, that will be added to the file; for example, the .txt or .py extensions.

file_dialog.setDirectory(str): This sets the current directory, specified in the parameter, for the file dialog.

file_dialog.setDirectory(QtCore.QDir): This sets the current directory, specified in the parameter, for the file dialog.

file_dialog.setDirectoryUrl(QtCore.QUrl): This sets the URL of the current directory, specified in the parameter, for the file dialog.

file_dialog.setFileMode(QtWidgets.QFileDialog.FileMode): This sets the file mode, specified in the parameter, for the file dialog. It describes the number and types of files that can be selected. The available modes are as follows:

  • QtWidgets.QFileDialog.AnyFile—0: Any filename.
  • QtWidgets.QFileDialog.ExistingFile—1: A single filename if it exists.
  • QtWidgets.QFileDialog.Directory—2: A directory name.
  • QtWidgets.QFileDialog.ExistingFiles—3: One or more filenames if they exist.

file_dialog.setFilter(QtCore.QDir.Filters): This sets the filter, specified in the parameter, that will be used to define which kind of files are to be shown.

file_dialog.setHistory([str]): This sets the browsing history of the file dialog as a list of paths specified in the parameter.

file_dialog.setIconProvider(QtWidgets.QFileIconProvider): This sets the icon provider, specified in the parameter, that will be used with the file dialog.

file_dialog.setItemDelegate(QtWidgets.QAbstractItemDelegate): This sets the item delegate, specified in the parameter, that will be used to render the items in the views of the file dialog.

file_dialog.setLabelText(QtWidgets.QFileDialog.DialogLabel, str): This sets the text (the second parameter) that will be shown in the label (the first parameter) in the file dialog.

file_dialog.setMimeTypeFilters([str]): This sets the filters, specified in the parameter, as a list of Multipurpose Internet Mail Extensions (MIME) types that will be used in the file dialog.

file_dialog.setNameFilter(str): This sets the filter, specified in the parameter, as a file type that will be used in the file dialog.

file_dialog.setNameFilters([str]): This sets the filters, specified in the parameter, as a list of the file types that will be used in the file dialog.

file_dialog.setOption(QtWidgets.QFileDialog.Option, bool): If the second parameter is True, then it sets the option (the first parameter) that will be used with the file dialog.

file_dialog.setOptions(QtWidgets.QFileDialog.Options): This sets the options that will be used with the file dialog.

file_dialog.setProxyModel(QtCore.QAbstractProxyModel): This sets the model for the views to the proxy model specified in the parameter.

file_dialog.setSidebarUrls([QtCore.QUrl]): This sets the URLs, specified in the parameter, for the sidebar of the file dialog.

file_dialog.setSupportedSchemes([str]): This sets the URL schemes that the file dialog will allow the user to navigate.

file_dialog.setViewMode(QtWidgets.QFileDialog.ViewMode): This sets the view mode, specified in the parameter, that will be used for displaying directories and files in the file dialog. The available modes are as follows:

  • QtWidgets.QFileDialog.Detail—0: The icon, name, and details for each item in the directory.
  • QtWidgets.QFileDialog.List—1: Only the icon and name for each item in the directory.

functional

These functions return the current values of the file dialog, inform you about changes to functionality, and more:

file_dialog.acceptMode(): This returns the accept mode of the QtWidgets.QFileDialog.AcceptMode type for the file dialog.

file_dialog.defaultSuffix(): This returns the default suffix added as a filename extension, if no other suffix is specified.

file_dialog.directory(): This returns the directory of the QtCore.QDir type that is displayed with the file dialog.

file_dialog.directoryUrl(): This returns the URL of the QtCore.QUrl type for the directory that is displayed with the file dialog.

file_dialog.fileMode(): This returns the file mode of the QtWidgets.QFileDialog.FileMode type that is used with the dialog.

file_dialog.filter(): This returns the filter of the QtCore.QDir.Filters type that is used when displaying the files in this dialog.

file_dialog.getExistingDirectory(QtWidgets.QWidget, str, str, QtWidgets.QFileDialog.Options): This creates a modal file dialog with the parent (the first parameter), a caption (the second parameter), the working directory (the third parameter), and options (the fourth parameter). It returns the directory that was selected. The available options are as follows:

  • QtWidgets.QFileDialog.ShowDirsOnly: File dialog shows only directories.
  • QtWidgets.QFileDialog.DontUseNativeDialog: File dialog as Qt standard.
  • QtWidgets.QFileDialog.DontResolveSymlinks: File dialog doesn't resolve symlinks.
  • QtWidgets.QFileDialog.DontConfirmOverwrite: File dialog doesn't ask for confirmation of an overwrite if an existing file is selected.
  • QtWidgets.QFileDialog.ReadOnly: Modal is read-only
  • QtWidgets.QFileDialog.HideNameFilterDetails: File dialog hides the filename filter details.
  • QtWidgets.QFileDialog.DontUseCustomDirectoryIcons: File dialog uses the default directory icon.

file_dialog.getExistingDirectoryUrl(QtWidgets.QWidget, str, QtCore.QUrl, QtWidgets.QFileDialog.Options, [str]): This creates a modal file dialog with the parent (the first parameter), a caption (the second parameter), the working local/remote directory (the third parameter), options (the fourth parameter), and the supported scheme (the fifth parameter) for the restriction of the URLs. It returns the directory of the QtCore.QUrl type that was selected.

file_dialog.getOpenFileName(QtWidgets.QWidget, str, str, str, str, QtWidgets.QFileDialog.Options): This creates a modal file dialog with the parent (the first parameter), a caption (the second parameter), the working directory (the third parameter), a filter (the fourth parameter) for these files, the selected filter (the fifth parameter) for these files, and options (the sixth parameter). It returns the file that was selected.

file_dialog.getOpenFileNames(QtWidgets.QWidget, str, str, str, str, QtWidgets.QFileDialog.Options): This creates a modal file dialog with the parent (the first parameter), a caption (the second parameter), the working directory (the third parameter), a filter (the fourth parameter) for these files, the selected filter (the fifth parameter) for these files, and options (the sixth parameter). It returns one or more files that were selected.

QPrinter

Many applications allow users to print documents or content that has been created. The Qt library has a basic dialog window to print created content. The QPrinter class is also available, which provides customization and operations with the default system printer device, as well as other features, such as printer settings. To understand the features of this class, it is recommended that you look at the official Qt documentation. Here, we will describe and implement the QPrintDialog and QPrintPreviewDialog classes. Both of these classes are included in the QtPrintSupport module. To use these classes in the application, we need to add the following to the u_app.py files.

Add this import statement in the u_app.py file of PyQt5:

...
from PyQt5 import QtPrintSupport
...

Additionally, add this import statement to the u_app.py file of PySide2:

...
from PySide2 import QtPrintSupport
...

QPrintDialog

This class provides a dialog window for selecting and configuring options and printing content. We can change the printing document settings, such as the number of copies. The declaration syntax of this class in the application is as follows:

print_dialog = QtPrintSupport.QPrintDialog()

The QPrintDialog class enhances the functionality with the functions described in the following sections.

set

These functions set the parameters and properties of the print dialog:

print_dialog.setOption(QtPrintSupport.QAbstractPrintDialog.PrintDialogOption, bool): If the second parameter is True, it sets the option (the first parameter) that will be enabled with this print dialog.

print_dialog.setOptions(QtPrintSupport.QAbstractPrintDialog.PrintDialogOptions): This sets the options, specified in the parameter, that will be enabled with this print dialog.

functional

These functions return the current values of the print dialog, inform you about changes to functionality, and more:

print_dialog.open(QtCore.QObject, str): This connects a signal to a slot with a receiver (the first parameter) and a member (the second parameter).

print_dialog.options(): This returns the options of the QtPrintSupport.QAbstractPrintDialog.PrintDialogOptions type that is used for this print dialog.

print_dialog.testOption(QtPrintSupport.QAbstractPrintDialog.PrintDialogOption): This returns True if the option specified in the parameter is enabled for this print dialog; otherwise it returns False.

signals

The following signals are available with the QPrintDialog class:

print_dialog.accepted(QtPrintSupport.QPrinter): This signal is emitted when the values set in the print dialog are accepted by the printer passed in the parameter.

QPrintPreviewDialog

Summary

In this chapter, we explored the basic principles of implementing dialog windows in GUI applications. Not all the dialogs in Qt have been covered here. Dialog windows such as QInputDialog, QProgressDialog (a dialog with a progress bar), and QWizard, which creates a framework for wizards in the app, can all be used. For printing, there is QPageSetupDialog, which is used to configure page-related options. Additionally, printing features can be extended with classes that do not represent the dialog windows, but they are very useful, such as QPrinterInfo and QPrintPreviewWidget.
